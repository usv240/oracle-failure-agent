How The Failure Oracle Works

An AI agent that matches your live startup metrics against a library of documented failure patterns — and fires an early warning before the crisis becomes fatal.

1

You enter your metrics

MRR, churn rate, burn rate, NPS, runway, LTV:CAC — the 11 key signals that determine a startup's trajectory. Or load a real historical scenario (Quibi, WeWork, Theranos) to see what the Oracle would have said.

2

Hybrid semantic search via MongoDB Atlas

Your metrics are converted into a text description and embedded using MongoDB Voyage AI voyage-4-large (1024-dim, asymmetric retrieval). Two searches run in parallel: Atlas Vector Search (cosine similarity) and Atlas Search (BM25 full-text) — both results merged via Reciprocal Rank Fusion for higher recall than either alone.

3

Vertex AI Gemini 2.5 Flash scores each candidate in parallel

Up to 5 candidate patterns are scored simultaneously by Vertex AI Gemini 2.5 Flash (thinking_budget=0 for speed). For each pattern, it evaluates metric-trigger alignment and returns a pattern similarity score (0–100%), detected warning signals, and estimated days to crisis. Parallel execution via asyncio.gather keeps total response time to 3–5 seconds.

Vertex AI Gemini 2.5 Flash
Parallel async scoring
4

⚖️ Challenger Agent independently verifies — adversarial AI

A second Gemini 3 instance is invoked with a deliberately skeptical prompt. Its job is to look for reasons the pattern does NOT apply — which healthy metrics contradict the match? It returns an independent pattern similarity score. If the two agents disagree by more than 10 percentage points, the result is flagged as DISPUTED and you see both scores side by side. This is dual-agent adversarial verification — not a single model's opinion.

If metrics are below every known pattern (<40% match): Uncharted Territory — honest uncertainty rather than a forced match.

Technology Stack

Vertex AI — Gemini 2.5 Flash (Scorer) Primary pattern scorer — parallel scoring of all candidates with thinking_budget=0 for maximum speed. Also handles decision auditing with thinking_budget=1024 for deeper reasoning. Why Vertex AI? Consistent latency SLA + enterprise billing.
Gemini 3 Flash (ADK Agents) All three ADK agents (Investigator, Challenger, Reporter) run on Gemini 3 Flash via paid Gemini API. Gemini 3 Flash is not yet available on Vertex AI, so agents route via the Gemini API key. This dual-model architecture is intentional: best model for each job.
MongoDB Voyage AI — voyage-4-large 1024-dimensional embeddings for all 100 patterns and every query. Asymmetric retrieval: patterns encoded as "documents," queries as "queries" — different encodings optimize for search vs. storage. Falls back to Google text-embedding-004 if Voyage is unavailable.
MongoDB Atlas — 6 Features Vector Search (cosine similarity) + Atlas Search (BM25 text) merged via Reciprocal Rank Fusion · $graphLookup cascade traversal · ACID multi-document transactions · Change Streams (Bayesian probability updates) · MCP server (tool-based reads) · $bucket/$facet cohort aggregations.
Google ADK — SequentialAgent Pipeline google-adk orchestrates 3 LlmAgents: Investigator (finds pattern) → Challenger (disputes it) → Reporter (synthesises + saves report). Each agent has dedicated FunctionTools and passes results via output_key through shared session state. Real-time events streamed via SSE + asyncio.Queue.
MongoDB MCP mongodb-mcp-server@1.9.0 — 28 tools, persistent stdio connection, handles all pattern reads/writes/aggregations in the critical path
Cloud Run Serverless deployment on Google Cloud with auto-scaling
MongoDB MCP Model Context Protocol server for direct agent database access
FastAPI + Motor Async Python backend with parallel Gemini calls for 2-4s responses

Where The Data Comes From

The 100-pattern failure library is synthesized from primary sources and encoded into MongoDB Atlas as 1024-dim vector embeddings. Every pattern has a citation trail and structured trigger thresholds.

YC Post-Mortems Paul Graham essays, YC partner office hours patterns, batch cohort analysis, "Default Alive or Default Dead" framework

VC Research Sequoia "RIP Good Times", Bessemer SaaS benchmarks, OpenView PLG reports, First Round Capital founder analysis, a16z essays
Public Post-Mortems Homejoy, Fab.com, Quibi, Jawbone, Juicero, Rdio, Zynga, Beepi — documented with specific metric timelines
SaaS Benchmarks David Skok SaaS metrics framework, Jason Lemkin SaaStr data, OpenView SaaS benchmark reports, Crunchbase funding patterns
Founder Frameworks Ben Horowitz "The Hard Thing About Hard Things", Marc Andreessen PMF essay, Eric Ries "The Lean Startup", Hamilton Helmer "7 Powers"
Each pattern's survival rate is calibrated against documented outcomes from the cited sources above — cross-referenced across multiple sources where possible. Warning signal timing is derived from post-mortem analysis of when each indicator first appeared relative to the failure event. The library is designed for directional signal detection: patterns flag structural risk; they do not constitute financial advice. Confidence scores reflect pattern similarity, not outcome probability.

Metrics Glossary — what do these numbers mean? Show ▾
MRR
Monthly Recurring Revenue — your total subscription revenue per month. If 100 customers pay $100/mo, MRR = $10,000.
Healthy early-stage: growing 15%+ per month
Monthly Growth Rate
How fast your MRR grows each month. Enter as decimal: 0.18 = 18% monthly growth. "Triple, triple, double, double" = ~15% monthly.
Good: >15% · Warning: 8–15% · Danger: <8%
Churn Rate
The % of customers who cancel each month. If 100 customers and 5 cancel, churn = 5% (enter 0.05). High churn means a leaky bucket — you acquire and immediately lose customers.
Good: <5% · Warning: 5–8% · Danger: >8%
Burn Rate
Net cash spent per month — your expenses minus revenue. If you spend $200K and earn $80K, burn = $120K/mo. This is how fast you're spending your funding.
Should decrease relative to MRR as you scale
Runway
Months until you run out of cash = Cash in bank ÷ Monthly burn. 8 months means you have 8 months to raise, grow to profitability, or cut costs.
Good: >18mo · Warning: 9–18mo · Danger: <9mo
NPS (Net Promoter Score)
Customer love metric (-100 to +100). Ask "How likely are you to recommend us?" Promoters (9-10) minus Detractors (0-6). Below 0 means more unhappy customers than happy ones.
Good: >50 · Warning: 30–50 · Danger: <30
CAC
Customer Acquisition Cost — total sales + marketing spend ÷ new customers acquired that month. If you spend $50K and get 10 customers, CAC = $5,000.
CAC should be <⅓ of LTV for healthy unit economics
LTV
Lifetime Value — total revenue you'll earn from one customer. Simple formula: Average monthly revenue per customer ÷ Monthly churn rate. $200/mo ÷ 5% churn = $4,000 LTV.
LTV:CAC ratio should be >3x for sustainable growth
Don't have exact numbers? Use your best estimates. Even rough numbers trigger meaningful pattern matches. The Oracle is designed for directional accuracy, not accounting precision.
